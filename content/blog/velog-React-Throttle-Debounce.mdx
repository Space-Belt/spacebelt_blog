> 프론트엔드 개발을 하다보면, 어떠한 사용자의 행동에 의해 엄청나게 많은 이벤트가 발생한다는 사실을 알 수있다. 그래서 특정 행동시에 일어나는 수많은 이벤트를 모두 다 처리하게 하는것이 효율적이지 않다는 생각을 했고, 이를 방지하기 위해 이벤트 제한 방법을 알아보았고 기록한다.
> 

먼저, **Throttle / Debounce** 에 대하여 간단히 알아보고, 따로 분류해서 알아보도록 하자.

가장 대표적으로, 사용자가 검색창에 검색어를 입력한다. 

![image](https://github.com/user-attachments/assets/47b13e75-d315-4a9a-a613-6025abe907c5)


- 사진에서 확인 가능하듯, 입력 하나하나  모두 console에 찍히는것을 확인 할 수 있다.
- 만약, 입력에 따라 실시간으로 검색된다면 사용자가 한 단어를 모두 입력하기 전에 계속해서 [ㄱ, 거, 검, 검ㅅ, 검새, 검색] 이런식으로 검색어 입력 과정에 따라 계속 api 호출이 이루어지게 된다.
- 이런식으로 작동한다면, 쓸모없는 api호출이 너무 많아지므로, 당연히 메모리낭비가 이루어지며, 비효율적인 검색 방식이 되는것이다.
- 대표적인 이벤트에는 **스크롤, 브라우저 리사이즈, 마우스 움직임,  버튼 중복 클릭** 등등이 있다.

이러한 이벤트들은 사용자의 움직임에 따라 계속 발생한다. 이 모든 이벤트가 일어날때 어떠한 일을 하는 함수, api 호출 등등이 계속 일어난다면, 결국 너무 많은 작업을 수행하게 되고, 이는 곧 기능 저하로 이어질 수 밖에 없는것이다.

즉, 스로틀링과 디바운싱은 이러한 문제를 해결하기 위해 등장한 ,이벤트 제어 방식이며 이벤트 핸들러의 실행 빈도를 제어하는 역할을 한다.

이제 이 두가지를 분류하여 알아보자!

### Throttle

---

스로틀링(Throttling)은 이벤트 핸들러가 일정 시간 간격으로만 실행되도록 제한하는 방법이다. 이는 특히 연속적으로 발생하는 이벤트의 처리 빈도를 조절하는데 사용한다.

**작동 원리**

스로틀링은 이벤트가 발생하면 일정 시간 동안 추가적인 이벤트 실행을 막는다. 즉, 200ms 간격으로 스로틀링을 설정하면, 첫 번째 이벤트가 실행된 후 200ms 동안은 추가 이벤트가 발생해도 무시한다.

**간단한 스로틀 함수 & 함수 설명**

```jsx
function throttle(콜백함수, 지연시간) {
  let timer;
  return function () {
    if (!timer) {
      timer = setTimeout(() => {
        콜백함수.apply(this, arguments);
        timer = undefined;
      }, 딜레이시간);
    }
  };
}
```

1. throttle 함수는 콜백함수와, 딜레이시간을  매개 변수로 받는다.
2. timer를 undefined로 변수 선언을 한다.
3. throttle함수는 새로운 함수를 반환한다. 즉, event에 이 스로틀 함수를 사용하면 실제 연결되는 함수는 여기서 반환하는 새로운 함수다.
4. 이 함수는 !timer 조건에 의해 timer가 true 일때만 작동한다. 따라서, 처음엔 자동으로 undefined이기 때문에 함수 내부의 조건문이 실행된다.
5. 조건문에 들어가면, timer에 setTimeout으로 실행될 함수와, 딜레이될 시간이 설정된다.
6. **`callback.apply(this, arguments);`** 지정된 시간이 지나면 원본 콜백 함수를 실행한다.
7. **`apply`**를 사용하여 원래의 **`this`** 컨텍스트와 인자들을 전달합니다.
8. timer에 다시  undefined를 할당하여, 다음 이벤트에 따른 함수 실행을 허용해준다.

**대표적으로 스로틀이 사용되는 경우**

**스크롤 이벤트 처리**

- 사용자의 현재 스크롤 위치를 기록하고 분석하는 로직이 필요하다면, 스로틀을 사용하여 제어할 수 있다.
- 무한 스크롤 - 사용자가 지정한 위치에 스크롤이 가까워 질때, 추가적인 콘텐츠가 있다면, 로드해주는 기능이다. 이때도, 스로틀을 사용하여, 서버 부하를 줄이고, 끊김없는 콘텐츠 로딩을 제공할 수 있다.

**창 크기 조정**

- 브라우저의 크기를 사용자가 조정하게 되면, 이때도 수많은 이벤트가 일어나게 된다.
- 창 크기를 조절하면, 레이아웃이 변화하는데, 이때 너무많은 레이아웃을 계산하지 않도록 하여, 브라우저의 부하를 줄여 준다.

### Debounce

---

디바운스는 연속적으로 발생하는 이벤트를 그룹화하여 마지막 이벤트만 처리하는 방식이다. 특정 시간 동안 추가 이벤트가 발생하지 않을 때까지 기다린 후 마지막 이벤트만 처리한다.

**작동 원리**

1. 이벤트 발생 시 타이머 설정
2. 설정된 시간 내에 새 이벤트 발생 시 이전 타이머 취소 및 새 타이머 설정
3. 설정된 시간 동안 추가 이벤트가 없으면 함수 실행

**간단한 디바운스 함수 & 함수 설명**

```jsx
function debounce(함수, 지연시간) {
  let timeout;
  return function (...args) {
    clearTimeout(timeout);
    timeout = setTimeout(() => {
      함수.apply(this, args);
    }, 지연시간);
  };
} 
```

1. 스로틀과 동일하게 함수와, 지연시간을 매개변수로 받는다.
2. 지정된 시간이 지나면 실행될 함수를 할당할 timeout 변수를 선언한다.
3. debounce 함수는 throttle과 동일하게 새로운 함수를 반환하고, 이 함수와 이벤트가 이어진다.
4. clearTimeout(timeout) 을 통해 timeout에 할당된 함수의 실행을 취소한다.
5. timeout에 setTimeout을 통해 지연시간이 지나면  실행될 함수를 할당한다.
6. **`함수.**apply(this, arguments);` 지정된 시간이 지나면 함수가 실행된다.

⇒ 이벤트가 계속 발송해서 debounce함수가 실행되고, 지연시간 내에 일어나는 이벤트가 계속 실행되면, clearTimeout으로 인해 그 이벤트에 대한 함수 실행이 취소된다. 결국엔, 마지막 이벤트가 일어났을때, 실행시키고 싶은 함수가 실행되는것이다.

**대표적으로 디바운스가 사용되는 경우**

**검색어 추천 & 실시간 검색 & 폼 유효성 검사**

- 추천할 검색어를 띄워줄때에, 모든 input 입력에 따른 추천 검색어를 띄워주게 되면, 검색어를 불러올때에도 계속해서 부하가 가해지기에 이를 방지할때 사용.
- 검색에서도 똑같이 계속해서 검색 api를 요청하면, 서버에 부하가 생기기에, 이를 방지한다.
- 인풋에 들어갈 유효성 검사도 동일하게, 계속해서 유효성 검사하는 함수가 실행되기에, 디바운스를 사용하여, 효과적으로 관리가 가능하다.

**버튼 중복 클릭 방지**

- 버튼을 여러번 클릭하는 경우가 있는 버튼 ⇒ 어차피 결과값이 같은데, 모든 요청을 받아들일 필요가 없다면, 마지막 버튼 클릭을 인식하여, 마지막 클릭에만 원하는 함수를 실행시켜 효율적으로 관리가 가능하다.

### Throttle과 Debounce의 차이점

---

- 스로틀은 원하는 일정시간마다, 이벤트를 처리하려고 사용하는 것이다. 즉, 이벤트가 계속 일어난다면, 원하는 주기마다 어떤 처리를 가능하도록 하는것이다.
- 디바운스는 이벤트가 계속 발생하더라도, 결국에 이 시간내에 일어난 마지막 이벤트에만 함수를 실행하여, 처리하는 방식이다. ⇒  연속된 호출 중 마지막 함수만 실행한다. 즉, 일정 시간 동안 추가 호출이 없을 때 한 번만 실행되는것이다.

결론, 스로틀과 디바운스 둘다 이벤트 처리를 제한하는 방식이다. 

스로틀은 지정시간내에 연속해서 이벤트가 일어나던 말던 지정한 시간 주기로 이벤트가 실행된다.

디바운스는 일정 시간 내에 이벤트가 일어나다가 추가적인 이벤트가 일어나지 않을때, 마지막 이벤트에 연결된 함수가 실행되는것이다.

따라서, 정확한 사용처에 맞게 사용하는것이 중요한것이므로, 적절히 잘 사용하도록 하자!!
