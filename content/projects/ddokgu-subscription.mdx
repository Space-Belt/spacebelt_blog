## 프로젝트 개요

똑구는 사용자가 이용 중인 여러 구독 서비스를 한곳에서 관리하는 앱입니다. 구독 서비스 등록, 월 구독료 확인, 결제일 대시보드, 알림함, 마이페이지를 통해 반복 결제를 놓치지 않도록 돕는 구조로 만들었습니다.

Expo와 React Native 기반으로 iOS/Android 앱 화면을 구현했고, 인증, 구독 등록, 대시보드, 알림처럼 사용자가 실제로 이동하는 주요 프론트엔드 흐름을 연결했습니다.

## 기술 스택과 구조

- Expo SDK 53, React Native 0.79, React 19, TypeScript
- Expo 기반 iOS/Android 앱 화면 구현
- React Query 기반 서버 상태 관리
- Zustand 기반 UI 상태 관리
- React Navigation 기반 화면 전환
- Firebase Analytics, Mixpanel 행동 데이터 추적
- SecureStore 기반 토큰 저장
- 공용 API client에서 401 응답 처리 통합

## 구현 포인트

### 인증 흐름

카카오, 구글, 네이버 소셜 로그인을 앱 흐름에 맞게 연결했습니다. 로그인 후 발급받은 토큰은 SecureStore에 저장했고, API 요청 중 401 응답이 내려오면 공용 API client에서 자동 로그아웃 처리를 수행하도록 정리했습니다.

네이버 로그인은 SDK 방식에서 OIDC Authorization Code Flow로 개선했습니다. 앱에서 authorization code를 확보한 뒤 accessToken과 idToken을 백엔드에 전달하는 구조로 바꾸면서, 콜백 URL과 idToken 확보 문제를 함께 해결했습니다.

### 푸시 알림

Expo push token을 발급받아 `/v1/users/me/devices`로 등록하는 흐름을 구현했습니다. 토큰이 아직 준비되지 않은 상태에서 빈 토큰이 서버로 전송될 수 있는 가능성을 제거해, 디바이스 등록 요청의 안정성을 높였습니다.

### 대시보드

결제일이 가까운 구독 서비스를 먼저 확인할 수 있도록 리스트를 구성했습니다. 구독 서비스가 5개를 초과하면 더보기/접기로 전환되며, height animation을 적용해 리스트가 갑자기 튀지 않고 자연스럽게 확장되도록 만들었습니다.

월 구독료 확인 카드, 소비 분석 카드, 결제일 D-day 배지, 무제한 배지처럼 구독 관리 앱에서 사용자가 자주 확인하는 정보를 빠르게 읽을 수 있도록 UI 밀도를 조정했습니다.

### 온보딩과 빈 상태

Figma 기준 이미지를 반영해 온보딩 1, 2, 3 화면을 구성했습니다. 기기 크기별 반응형 레이아웃을 적용했고, 작은 화면인 iPhone SE에서도 이미지와 버튼이 겹치지 않도록 대응했습니다.

알림함은 알림이 없을 때 빈 화면이 어색하게 보이지 않도록 `ListEmptyComponent` 기반의 empty state를 추가했습니다. 빈 상태 UI도 실제 기능 흐름의 일부로 보고, 사용자가 다음 행동을 이해할 수 있게 정리했습니다.

마이페이지에는 관리자 문의 모달을 추가했습니다. 제목, 내용, 사진 첨부 UI를 제공해 사용자가 앱 안에서 바로 문의 내용을 작성할 수 있도록 구성했습니다.

- 네이버 로그인 Callback URL 설정 문제

- Expo push token 발급 타이밍 문제

## 협업하며 개선한 점

PR 리뷰를 반영하면서 공용 컴포넌트의 변경 범위를 줄이고, 각 화면 컴포넌트가 맡는 책임을 분리하는 방향으로 수정했습니다. 공통 API client, 인증 처리, 서버 상태 관리처럼 여러 화면에 영향을 주는 부분은 한곳에서 처리되도록 정리해 유지보수 부담을 줄였습니다.

## 배운 점

React Native 앱은 화면 구현만으로 끝나지 않았습니다. 소셜 로그인, SecureStore, 푸시 토큰, API 예외 처리, 작은 기기 대응까지 모두 연결되어야 실제 사용 가능한 앱이 되었습니다.

특히 인증과 푸시 알림처럼 앱 바깥의 플랫폼 정책과 백엔드 계약이 맞물리는 기능은, UI보다 흐름의 안정성이 더 중요했습니다. 이번 프로젝트를 통해 서버 상태와 UI 상태를 분리하고, 공통 client에서 예외 흐름을 처리하는 구조가 앱 유지보수에 얼마나 중요한지 확인했습니다.
